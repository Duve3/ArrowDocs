## Pull requests

We accept basically any pull request as long as it passes these simple things:
- Adds genuine new, teachable content **OR** fixes a critical issue
- Builds (trust me, you'd be surprised)
- Doesn't remove content without a very specific reason listed in the PR
- Docs are reasonably sized - sometimes its better to have many shorter docs than having some really long ones! (this is case by case)

Even if you fail to meet any of those, we will work with you to help you fix these things and make our documentation better than it ever has been!
