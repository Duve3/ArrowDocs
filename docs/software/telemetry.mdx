# Telemetry

What is telemetry? Well telemetry is all that data you see in something like Elastic or AdvantageScope.

Telemetry might seem simple, its just sending some data back to the computer every periodic, we can do this easy peazy!

But its so much more...

There are two kinds of telemetry, theres `one-way telemetry`, and `two-way telemetry` (these are my personal terms, I don't know if anyone actual uses these)

`One-way telemetry` is on the simpler side, you just gotta publish some data using either `NetworkTables` or `SmartDashboard`.
This is what you would use for something like live swerve drive states, or live motor input values, etc. It's information that you cannot change, but should report.

`Two-way telemetry` is where it becomes more complicated. In this form, you need to be able to communicate both ways (from the robot to the laptop, and from the laptop to the robot).
You'll see this used for things like AutoChoosers, RobotModes, etc.

Now specifically I will focus on AutoChooser, it's something that every single team needs, no matter what. You need a way to create a dropdown that allows your team to pick what auto to run.

AutoChoosers may seem complicated, its an entirely new thing to learn and understand. But they are relatively simple
