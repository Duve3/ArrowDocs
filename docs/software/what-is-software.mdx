# What is software even about??

Software is one of the most crucial parts of robot creation and development. 

The hardware on your robot will define your limits, but the software will decide whether you hit that limit or not -
Your hardware could be the greatest CAD ever made, but if the software is lacking, you will never hit the maximum possible out of your robot.

At ArrowDynamics, all software is written in Java. Learning and understanding java is crucial, heres some good resources:
- [CodeAcademy](https://www.codecademy.com/learn/learn-java) - more interactive, requires a free account
- [LearnJavaOnline](https://www.learnjavaonline.org/) - less interactive, but no account or anything
- [LearnCSOnline](https://www.learncs.online/) - Select Java, multiple FRC Teams have reported this being a good resource
- Google - this is the greatest tools for all programmers, google always has some kind of solution (in the modern day, AI is often of similar or higher value)

<span className="subtext">If you have fundamentals of programming (i.e. advanced understanding of another programming language) it is more than possible to avoid taking the java course, but remember that it is highly recommended!</span>

## Do not overly trust AI
Currently in software globally, AI is hitting. Naturally you will want to use AI, and I do recommend it, but you should only use it in a certain way.

You should use AI as a tool to help you learn, as a hyper-specific teacher, not as someone who will spit out the correct code everytime.
Because with FRC coding, it's often extremely wrong (due to the way the AI are trained, they struggle with more medium level topics).
<span className="subtext">Team 4144 has reported high success rates with almost full AI development cycles - https://2026.team4414.com/#software - but they are "gatekeeping" what exactly they are doing (they only write a broad overview)</span>


Understanding WHAT you (or ChatGPT, Gemini, Claude, etc.) are writing is **exceptionally** important as a developer, it's a skill that will follow you beyond FRC and Java, but into your life.

A good check is if you had to explain this code to someone, could you do it? If you couldn't, then you need to chit chat with the AI or with someone about what exactly your code is doing.

It is very important that throughout the code writing; you either avoid AI, or learn the right way. If you mess up on these fundamentals, it will come back to hurt you later.

# Useful resources
Of course this place of documentation is great for software, but some of the following things are also quite useful:
- [FRC Software Docs](https://docs.wpilib.org/en/stable/docs/software/what-is-wpilib.html) - info about WPILib and actual FRC docs
- [WPILib JavaDoc](https://github.wpilib.org/allwpilib/docs/release/java/) - JavaDoc is basic documentation on every function, class, and variable in WPILib
- [CTRE Phoenix 6 Documentation](https://v6.docs.ctr-electronics.com/en/stable/) - Documentation specific to CTRE Phoenix 6 motors (which are the motors we use at ArrowDynamics)
- [REVLib Code Examples](https://github.com/REVrobotics/REVLib-Examples) - REVLib is used for Spark Maxes, Neos, and other REV related products (these motors are often used by beginner teams, and understanding them is also important)

:::note

While there are many types of motors, often the fundamentals from one motor will transfer - all that changes is how you write the code for it.

(i.e.: both Neos and Krakens have closed loop control, you just code each them slightly differently)

:::
